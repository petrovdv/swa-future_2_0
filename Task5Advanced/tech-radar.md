
# Технический радар

## Статусы принятия
| Статус     | Определение                                                                     |
| ---------- | ------------------------------------------------------------------------------- |
| **Adopt**  | Рекомендовано к использованию, является стандартом для внедрения во все домены  |
| **Trial**  | Пилотируется в 1–2 доменах, оценивается производительность и стоимость владения |
| **Assess** | Изучается, требует PoC, не готов к критическим нагрузкам                        |
| **Hold**   | Вывод из эксплуатации или запрет на новое внедрение                             |

## 1. Архитектурные паттерны и практики
| Паттерн / Практика                                  | Статус | Обоснование                                                                                                                            |
| --------------------------------------------------- | ------ | -------------------------------------------------------------------------------------------------------------------------------------- |
| **Data Mesh**                                       | Adopt  | Декомпозиция данных по доменам (Клиники, Финтех, ИИ, Партнёры). Витрина формируется через самообслуживание и продуктовый подход.       |
| **Event-Driven Architecture (EDA)**                 | Adopt  | Асинхронное взаимодействие доменов через шину событий. Устраняет синхронные зависимости Camel/DWH.                                     |
| **CQRS (Command Query Responsibility Segregation)** | Adopt  | Разделение write-моделей доменов и read-моделей витрины. Обеспечивает near-real-time аналитику без нагрузки на OLTP.                   |
| **Schema-First Development**                        | Adopt  | Контракты событий версионируются через Schema Registry. Обратная совместимость — обязательное требование.                              |
| **Anti-Corruption Layer (ACL)**                     | Adopt  | Изоляция легаси (SQL 2008, Camel) от новых доменов. Трансляция старых форматов в событийную модель.                                    |
| **Self-Service BI / Data Product**                  | Adopt  | Бизнес-аналитики работают с готовыми витринами. Конструктор отчётов без прямого доступа к raw-данным.                                  |
| **Локализация данных и соответствие 152-ФЗ**        | Adopt  | Архитектура изначально проектируется под хранение и обработку данных на территории РФ, с сегментацией контуров ПДн/медицинских данных. |
| **Event Sourcing**                                  | Hold   | Избыточен для текущего масштаба. CQRS + Durable Stream покрывают требования аудита и восстановления состояния.                         |
| **Centralized Monolithic DWH**                      | Hold   | Выводится из эксплуатации. Оставляет только исторический архив на этапе миграции.                                                      |

## 2. Платформы и инфраструктура
| Технология                                                       | Статус      | Обоснование                                                                                                                                                                        |
| ---------------------------------------------------------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **S3-совместимое Object Storage**                                | Adopt       | Единое дешёвое хранилище для сырых событий, лога и архивов. Используются Yandex Object Storage, Selectel Cloud Storage, VK Cloud S3. Соответствует требованиям локализации данных. |
| **Управляемое аналитическое хранилище (ClickHouse / Greenplum)** | Trial/Adopt | ClickHouse (Yandex Managed Service) — Adopt для витрин и агрегаций. Greenplum / Postgres Pro Enterprise — Trial для сложных MPP-запросов.                                          |
| **Event Streaming (Apache Kafka)**                               | Adopt       | Высокая пропускная способность, DLQ, ретайн событий. Развёртывается в Yandex Cloud / Selectel / VK Cloud или self-hosted.                                                          |
| **Stream Processing (Apache Flink / Spark Streaming)**           | Trial       | Для real-time агрегаций и материализации витрин. Оценивается сложность поддержки в РФ-кластерах vs готовые решения облачных провайдеров.                                           |
| **Schema Registry (Apicurio / Open-Source Kafka)**               | Adopt       | Обязательный компонент для валидации контрактов, версионирования и обратной совместимости. Confluent заменён на open-source/локальные сборки.                                      |
| **PowerBuilder / SQL Server 2008 ETL**                           | Hold        | Legacy. Поддержка только через ACL до полного вывода из контура.                                                                                                                   |

## 3. Аналитика, каталог и безопасность
| Компонент                                                           | Статус | Обоснование                                                                                                                                                             |
| ------------------------------------------------------------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Data Catalog & Governance (DataHub / Amundsen)**                  | Trial  | Автоматическое обнаружение метаданных, lineage, управление доступом (RBAC/ABAC). Критично для Data Mesh.                                                                |
| **Self-Service BI (Yandex DataLens / Apache Superset / Visiology)** | Adopt  | Конструктор дашбордов, семантический слой, row-level security. DataLens интегрируется с отечественным стеком, Superset/Visiology — для сложных корпоративных сценариев. |
| **CDC Engine (Debezium / Airbyte)**                                 | Adopt  | Бесшовная репликация легаси SQL 2008 в событийную шину на этапе миграции.                                                                                               |
| **Secrets & KMS Management (Yandex Lockbox / VK KMS / Vault)**      | Adopt  | Шифрование данных в rest/transit. Управление ключами для изоляции ПДн и финансовых транзакций. Интегрируется с СКЗИ для ГОСТ-алгоритмов.                                |
| **СКЗИ и криптография по ГОСТ**                                     | Adopt  | КриптоПро CSP, VipNet, ViPNet Coordinator. Обязательное требование для защиты медицинских и финансовых данных (152-ФЗ, ГОСТ 34.12/34.10).                               |
| **Power BI Desktop / Looker (локальные лицензии)**                  | Hold   | Замена на облачный Self-Service BI с управляемым доступом, аудитом и локализацией контура данных.                                                                       |
